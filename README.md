# InsuranceApp-SpringBoot（損保業務支援アプリ：Spring Boot / Web(MVC)版）

損保業務（自動車保険）を題材にした **保険担当者向けのミニ業務支援Webアプリ** です。  
契約更新の手続き漏れ防止（要注意期間の可視化）と、早期更改率の可視化を目的に、段階的に機能を追加していきます。

- **対象者**：保険代理店／保険会社の担当者（自分用のタスク支援）
- **UI**：Thymeleaf で画面（一覧／詳細／操作）を提供
- **DB**：MySQL
- **外部連携（予定）**：Googleカレンダー（満期7日前通知のトグル）

---

## リポジトリ構成と位置づけ

本リポジトリは **「最終成果物（本体）を育てる」** ためのリポジトリです。

- **100日チャレンジ（学習ログ／Dayまとめ）**  
  `100days-coding-challenge-java`（毎日更新・継続力の証跡）
- **Servlet/JSP版（完成版として公開済みの成果物）**  
  `InsuranceApp-ServletJSP`
- **Spring Boot版（本リポジトリ）**  
  `InsuranceApp-SpringBoot`（本体を日々改修して完成度を上げる）

## MVP（このアプリで最初に目指す範囲）

設計書（詳細仕様）に基づく、MVPの主な範囲です。

### Policy（契約）
- 一覧・詳細
- 更新（renew）／更新取消（当日限定）
- 解約（cancel）／解約取消（当日限定）
- **要注意期間（満期20日前〜満期日）** の可視化（バッジ等）
- **早期更改率（当年度・当月）** の表示
- **Googleカレンダー登録（7日前通知）** のトグル（予定）

### AccidentReport（事故）
- 一覧・詳細
- **滞留（7日以上未対応）** の可視化

> 補足：AI（要約／次アクション候補）は「画面表示のみ（保存しない）」として設計書上は想定されていますが、実装は段階的に検討します。

## 設計メモ（重要な業務ルールの例）

詳細は `doc/design/pdf/sonpo_app_detailed_spec_v1.pdf` を参照してください。  
READMEには「実装時に効くポイント」だけを抜粋します。

- **更新可能期間**：満期2か月前〜満期日
- **早期更改期間（推奨）**：満期2か月前〜満期21日前
- **要注意期間（最重要）**：満期20日前〜満期日（バッジ表示）
- **更新取消／解約取消**：当日限定（操作ミスの巻き戻し想定）
